The primary challenge addressed throughout these projects was the development of a high-performance system capable of managing and retrieving academic course data for a university curriculum. This problem required a solution that could ingest potentially large and unformatted datasets from external files and provide users with an organized, searchable interface.

In approaching this problem, I focused on the implementation of a Binary Search Tree (BST). Understanding data structures is essential because they dictate computational efficiency; while a simple list might suffice for small datasets, it becomes unmanageable as information scales. By choosing a tree-based structure, I aimed to ensure that the program could perform search and insertion operations in logarithmic time (O(logn)). However, I recognize that a standard BST is sensitive to the order of data insertion; if data is loaded in a sorted manner, the tree can become "unbalanced," causing performance to degrade to linear time (O(n)). To maintain true scalability in a production environment, a self-balancing tree (like an AVL or Red-Black tree) would be a logical next step.

Throughout the development process, I encountered roadblocks related to data cleaning and the validation of complex course relationships. Specifically, I had to ensure the program could handle hidden formatting characters within CSV files, which was a common real-world hurdle. I overcame this by engineering a custom tokenization process that stripped away non-visible characters.

I found the most rewarding challenge to be the prerequisite validation. I developed a post-load validation algorithm that performs a recursive traversal of the entire tree to identify and report any missing prerequisites. For example, by implementing a check like if (tree.search(prereqID) == nullptr), I was able to ensure that the system wouldn't just store data, but would actively maintain its logical integrity.

This project has significantly expanded my approach to software design by emphasizing the importance of algorithmic cost. Instead of merely focusing on whether a program produces the correct output, I now prioritize long-term performance. This experience taught me to evaluate software based on how it handles growth, shifting my mindset toward building systems that are robust enough to withstand real-world data volumes without requiring future restructuring.

Finally, this project has evolved my programming practice toward craftsmanship. By utilizing modular class structures and writing defensive code that validates data at the point of entry, I’ve created a system that is not only efficient but also maintainable and adaptable for any developer who might work on it in the future.
